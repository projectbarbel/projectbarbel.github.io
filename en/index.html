Bitemporal data management

Barbel Histo Core is a bitemporal in-memory data storage API. It keeps track of two time dimensions: effective time and record time.

In-memory transactions

Barbel Histo Core provides in-memory transaction isolation to support business critical applications.

Powered by cqengine

Built on Google's ultra-fast collection framework, Barbel Histo allows high-end performance batch processing operations.

Gets you started in minutes

Barbel core API can be used in various flavours. As full data storage API for microservices, as well as simple user-friendly API to manage your bitemporal data. All functions are provided by a simple API.

Built-in persistence support

Although it's not mandatory to operate with persistence storage backends, an SPI persistence provider endpoint is build-in. This allows to use Barbel Histo Core as single Pojo-based persistence layer.
